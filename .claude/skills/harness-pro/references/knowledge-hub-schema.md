```markdown
# Module: billing

## Responsibility
Manages user account balances — top-ups, deductions, and balance queries.
All amounts are stored in **cents (integer)** and converted to dollars only
in the display layer.

## Public Interface

### calculateBalance(userId: string): Promise<integer>
- **Intent**: query the current balance for a given user
- **Input**: userId — unique user identifier
- **Output**: balance in cents (integer)
- **Side effects**: none (read-only)
- **Note**: non-existent userId returns 0, does not throw

### deductBalance(userId: string, amount: integer): Promise<boolean>
- **Intent**: deduct from a user's balance
- **Input**: userId; amount in cents (must be a positive integer)
- **Output**: true = success, false = insufficient balance
- **Side effects**: writes to database, writes to operation log
- **Note**: negative amount throws InvalidAmountError

## Dependencies
- shared/db (database connection)
- shared/logger (operation logging)

## Depended on by
- api/routes/billing (HTTP routing layer)
- worker/settlement (scheduled settlement task)

## Known Technical Debt
- TD-003 ✅ resolved: float precision issue (fixed 2024-01-15)
- TD-015 🔴 open: circular dependency with user-profile module
```
