## 硬件现状（已全部连接至电脑）

### 主控制器 - STM32F411CEU6
- **MCU**: ARM Cortex-M4, 100MHz
- **连接**: 已通过 **ST-Link** 接入电脑
- **职责**: 飞控算法、电机控制、传感器数据采集

### 通信模块 - ESP32-C3
- **职责**: WiFi 通信，通过 UART 与 STM32 互联
- **连接**: 已接入电脑
- **WiFi配置**: SSID=`whc`, 密码=`12345678`

### 传感器
- **IMU**: ICM-42688-P (6轴陀螺仪/加速度计, SPI3)
- **气压计**: LPS22HBTR (I2C1)
- **磁力计**: QMC5883P (I2C1)

### 动力系统
- **电机**: 空心杯电机 × 4 (MOSFET驱动)
- **电源**: USB供电用于开发（安全模式，动力不足无法起飞）

### 硬件可靠性声明
**硬件无问题，已被验证过。如有问题一定是代码的问题。**

## 开发路线（每个阶段必须可烧录、可验证）

| 阶段 | 内容 | 验证方式 |
|------|------|----------|
| 1 | 传感器驱动（陀螺仪、气压计、磁力计） | 串口输出原始数据 |
| 2 | PWM 空心杯电机控制 | 示波器/万用表测 PWM 波形 |
| 3 | 串口通信（STM32↔ESP32、各自↔PC） | 回环测试/串口助手 |
| 4 | WiFi 通信（ESP32 作为 STA 连接 `whc` 网络） | ping/网络调试 |
| 5 | 引入 FreeRTOS | 任务调度验证 |
| 6 | 运动控制算法（AHRS + PID） | 姿态解算输出验证 |

---

## 关键约束

1. **0 人工参与**: AI 独立完成所有开发、烧录、调试
2. **每个模块必须设计验证步骤**: 代码必须可编译、可烧录、可调试
3. **调试驱动**: 通过 printf、逻辑分析等手段确认功能正确
4. **渐进迭代**: 小步快跑，每个阶段可独立验证
5. **安全优先**: USB供电用于开发（动力不足无法起飞）
