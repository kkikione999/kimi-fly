# 开发原则 - 渐进式披露

> **创建日期**: 2026-03-18
> **适用范围**: 无人机飞控项目全生命周期

---

## 1. 渐进式披露原则

### 1.1 核心定义

渐进式披露原则指：信息根据开发阶段的需求逐步公开，而非一次性全部暴露。这确保每个阶段都有清晰、可管理的上下文。

### 1.2 信息披露层级

| 阶段 | 披露内容 | 目的 |
|------|----------|------|
| **架构设计** | 高层模块划分、接口契约 | 建立整体框架 |
| **模块开发** | 模块内部实现细节、寄存器配置 | 专注当前实现 |
| **集成测试** | 跨模块交互、时序关系 | 验证集成正确性 |
| **调试验证** | 完整寄存器映射、信号波形 | 问题定位 |

### 1.3 实践准则

1. **按需获取**: 只提供当前任务所需的信息
2. **分层抽象**: 上层模块不依赖下层实现细节
3. **渐进深入**: 从概念到实现逐步细化
4. **上下文隔离**: 每个任务有独立的上下文边界

## 2. 硬件文档真实性条款

### 2.1 真实性声明

**以下文档为真实硬件设计文档，包含完整的元器件细节：**

| 文档 | 路径 | 内容 |
|------|------|------|
| **机身电路图** | `hardware-docs/SCH_机身_1-P1_2026-03-11.pdf` | 电机驱动、充电管理、USB-HUB、ST-Link |
| **主控电路图** | `hardware-docs/SCH_主控_1-P1_2026-03-11.pdf` | STM32F411、传感器(IMU/气压计/磁力计)、WiFi模块 |

### 2.2 元器件信息来源

所有元器件细节以PDF电路图为**唯一权威来源**：

- **芯片型号**: 以电路图标注为准 (如 ICM-42688-P, LPS22HBTR, QMC5883P)
- **引脚连接**: 以电路图连线为准
- **电源网络**: 以电路图网络标号为准 (3V3_D, 3V3_IMU, 3V3_RF, V_BAT)
- **阻容参数**: 以电路图标注为准

### 2.3 使用约束

1. **禁止推测**: 不得基于经验推测元器件参数，必须查阅电路图
2. **版本锁定**: 引用电路图时必须注明版本日期 (2026-03-11)
3. **变更追踪**: 任何与电路图不符的发现必须记录为技术债务
4. **交叉验证**: 引脚定义(`hardware-docs/pinout.md`)必须与电路图保持一致

### 2.4 信息获取流程

## 3. 渐进式披露与真实性的平衡

### 3.1 当前阶段披露策略

| 开发阶段 | 硬件信息披露深度 | 参考文档 |
|----------|------------------|----------|
| **Phase 1: HAL层** | 引脚编号、GPIO模式 | `hardware-docs/pinout.md` |
| **Phase 2: 传感器驱动** | I2C/SPI时序、寄存器地址 | 电路图 + 芯片手册 |
| **Phase 3: 姿态解算** | 传感器数据范围、采样率 | 芯片手册 |
| **Phase 4: 飞控算法** | 电机PWM特性、动力参数 | 电路图 + 电机规格 |
| **Phase 5: 通信协议** | WiFi模块AT指令、波特率 | 电路图 + 模块手册 |

### 3.2 信息检索优先级

1. **第一优先级**: 电路图PDF (`hardware-docs/SCH_*.pdf`)
2. **第二优先级**: 引脚定义 (`hardware-docs/pinout.md`)
3. **第三优先级**: 芯片数据手册 (外部文档)
4. **第四优先级**: 项目历史代码

## 4. 违规处理

### 4.1 常见违规

| 违规类型 | 示例 | 纠正措施 |
|----------|------|----------|
| **凭经验推测** | "ICM-42688 应该跟MPU6050一样" | 查阅真实电路图确认 |
| **参数假设** | "SPI时钟随便设个1MHz" | 查阅芯片手册最大时钟 |
| **引脚混淆** | "I2C1在PB6/PB7"(未确认) | 核对电路图引脚标注 |

### 4.2 发现不一致时

1. 停止当前工作
2. 核对电路图PDF
3. 记录差异到 `docs/plans/tech-debt-tracker.md`
4. 更新 `hardware-docs/pinout.md` 以反映真实硬件
5. 继续开发
