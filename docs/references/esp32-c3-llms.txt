# ESP32-C3 Reference Materials for LLMs

ESP32-C3 开发参考资料，供 Agent 查询使用。

## 芯片概览

### Specifications
- Core: RISC-V RV32IMC(F) (single-core)
- Clock: Up to 160 MHz
- SRAM: 400 KB
- Flash: External, up to 16 MB
- WiFi: 802.11 b/g/n
- BLE: 5.0

### Memory Map
- IRAM: Instruction RAM
- DRAM: Data RAM
- RTC FAST Memory: 8 KB (retained in sleep)
- RTC SLOW Memory: 8 KB (retained in sleep)

## ESP-IDF Framework

### Key Components
- FreeRTOS: RTOS kernel
- HAL: Hardware abstraction
- Driver: Peripheral drivers
- WiFi/BT: Wireless stack
- NVS: Non-volatile storage

### Project Structure
```
project/
├── main/
│   ├── CMakeLists.txt
```

## Common Patterns

### Task Creation
```c
xTaskCreate(task_function, "task_name", stack_size, param, priority, &handle);
```

### GPIO
```c
gpio_set_direction(GPIO_NUM_X, GPIO_MODE_OUTPUT);
gpio_set_level(GPIO_NUM_X, 1);
```

### UART
```c
uart_config_t config = {
    .baud_rate = 115200,
    .data_bits = UART_DATA_8_BITS,
    .parity = UART_PARITY_DISABLE,
    .stop_bits = UART_STOP_BITS_1,
    .flow_ctrl = UART_HW_FLOWCTRL_DISABLE
};
uart_param_config(UART_NUM_1, &config);
uart_driver_install(UART_NUM_1, rx_buf_size, tx_buf_size, 0, NULL, 0);
```

### WiFi
```c
// Station mode
esp_wifi_set_mode(WIFI_MODE_STA);
esp_wifi_set_config(WIFI_IF_STA, &wifi_config);
esp_wifi_start();
```

## Power Management

### Sleep Modes
- Active: Full operation
- Modem-sleep: CPU on, WiFi/BT off
- Light-sleep: CPU paused, RTC on
- Deep-sleep: Only RTC memory retained

### Wake Sources
- GPIO
- Timer
- UART
- Touch (not available on C3)

## Debugging

### JTAG
- GPIO4: MTCK
- GPIO5: MTDI
- GPIO6: MTMS
- GPIO7: MTDO

### Logging
```c
ESP_LOGI(TAG, "Info message: %d", value);
ESP_LOGW(TAG, "Warning message");
ESP_LOGE(TAG, "Error message");
```

## Security

### Features
- Secure Boot V2
- Flash Encryption
- Digital Signature
- HMAC

### Best Practices
- Enable secure boot in production
- Encrypt flash contents
- Use secure boot with flash encryption

## Common Issues

### WiFi Connection
- Check power supply (brownout causes issues)
- Verify antenna connection
- Check channel and country code

### Memory
- Stack overflow: Increase stack size or reduce usage
- Heap exhaustion: Check for leaks, reduce allocations
- IRAM/DRAM placement: Use IRAM_ATTR for ISR code

### Build
- Clean build after sdkconfig changes: `idf.py fullclean`
- Check component dependencies
- Verify CMakeLists.txt syntax
