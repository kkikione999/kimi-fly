# Task 301: ICM42688 I2C 地址修正 (0x68 → 0x69)

## 目标
修正 ICM42688 驱动的 I2C 地址为 0x69，与实际硬件 AD0 引脚拉高状态一致。

## 背景上下文

### 相关代码
- 文件: `firmware/stm32/drivers/icm42688.h` - 当前地址宏 `ICM42688_I2C_ADDR = 0x68`

### 问题说明
硬件验证结果：IMU 响应地址为 0x69（AD0 引脚被拉高至 VCC）。
代码中硬编码 0x68，导致与实际硬件不符。

### 依赖关系
- 前置任务: 无
- 外部依赖: 无

### 硬件信息
- 涉及引脚: PB6 (I2C1_SCL), PB7 (I2C1_SDA)
- I2C 地址: 当前代码 0x68，实际硬件 0x69 (AD0=VCC)
- 参考: `hardware-docs/pinout.md` 第 3.1 节

### 代码规范
- 只修改宏定义，不改逻辑
- 同步更新注释

## 具体修改要求

### 文件 1: `firmware/stm32/drivers/icm42688.h`
1. 将 `ICM42688_I2C_ADDR` 从 `0x68U` 改为 `0x69U`
2. 更新文件头注释和宏注释，说明 AD0=VCC 故为 0x69
3. 保留原注释 `@note AD0=GND -> 0x68, AD0=VCC -> 0x69` 作为说明
